# Приложение для закладок (link manager)

Простой веб-приложение на Flask и SQLite для сохранения закладок: регистрация пользователей, вход в систему, добавление/удаление ссылок с комментариями.

## Возможности
- Регистрация пользователей (имя пользователя + пароль).
- Хранение паролей в виде SHA-256 хэша.
- Управление пользовательскими ссылками: заголовок, URL, комментарий.
- Удаление своих закладок.
- Сессии для аутентификации пользователей.

## Требования
- Python 3.x  
- Flask  
- SQLite3 (входит в стандартную библиотеку Python)

## Установка и запуск
1. Клонируйте репозиторий и перейдите в папку проекта:
   ```bash
   git clone https://github.com/your-repository-url.git
   cd your-repository-name
   ```

2. Установите зависимость:
   ```bash
   pip install Flask
   ```

3. Запустите приложение:
   ```bash
   python app.py
   ```

4. Откройте в браузере:
 http://192.168.1.15:5000

При первом запуске автоматически создаётся файл базы данных database.db и нужные таблицы.

## Структура кода
- app.py — основной файл приложения с маршрутами, логикой авторизации и работы с БД.
- template/ — папка с HTML-шаблонами (register.html, login.html, index.html).
- static.css — (путь указан как static_folder в приложении) — стили.
- database.db — файл SQLite (создаётся автоматически).

## Схема базы данных
1. Таблица users:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - username TEXT UNIQUE
   - password TEXT (SHA-256 хэш)

2. Таблица links:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - title TEXT
   - url TEXT
   - comment TEXT
   - user_id INTEGER (ссылка на users.id)

## Использование
1. Регистрация: перейти на /register и создать учётную запись.
2. Вход: /login — после успешного входа перенаправление на главную страницу.
3. Добавление закладки: на главной странице заполнить поля title, url, comment и отправить форму.
4. Просмотр/удаление: на главной странице отображаются только закладки текущего пользователя; удалить можно по ссылке удаления.
5. Выход: /logout очищает сессию и возвращает на страницу входа.

## Важные замечания по безопасности
- Пароли хранятся в хэше SHA-256 — лучше использовать соль и алгоритмы вроде bcrypt/argon2 для производства.
- app.secret_key задан как 'secret123' — в реальном развертывании используйте случайный, секретный ключ и храните его вне репозитория (переменные окружения).
- В текущем коде нет защиты от CSRF и ограничений на ввод URL/текста — для продакшена добавьте валидацию и CSRF-защиту.
